# Cartographer 涵蓋什麼 / 不涵蓋什麼

> 先讀這份，避免誤用。Cartographer 是工具鏈的一環，刻意不做別人做的事。

---

## ✅ 涵蓋

| 範疇 | 說明 |
|---|---|
| 從零起草軟體 PRD | 逐節訪談把模糊想法變成合格 PRD（15 模組） |
| 審查/補強既有 PRD | 把品質閘反過來當體檢清單，找缺漏並給修補建議 |
| 需求品質強制 | 原子＋編號＋優先級＋可驗收＋可追溯；prd_lint 機械化把關 |
| 非功能 / 資安 / 隱私 | 強迫面對效能、SLA、可觀測、無障礙、i18n、資料分類、法遵 |
| 可追溯矩陣 | FR ↔ persona ↔ 目標 ↔ AC ↔ 里程碑 |
| 交棒 compass | 轉成 compass 能吃的 checklist 與反向審計輸入 |

## ❌ 不涵蓋（交給誰）

| 不做 | 交給 |
|---|---|
| 實作 PRD、照規格施工、不偏航 | **Compass** |
| 怎麼想、根因診斷、資安思考習慣 | **Sentinel** |
| 硬體 PRD（機構、電性、製造、法規測試） | 本 skill 專注軟體；硬體請另尋 |
| 商業計畫書 / 財務模型 / 募資簡報 | 非 PRD 範疇 |
| 詳細 UI 設計稿 / wireframe / 視覺規範 | 設計工具；PRD 只描述行為與狀態 |
| 專案管理排程細節（甘特圖、工時） | PM 工具；PRD 只到里程碑與切片 |

## 🤝 與 Sentinel / Compass 的接縫

- **進來之前**：有想法但混亂 → Sentinel 幫想清楚 → Cartographer 開始起草。
- **§08 資安**：直接引用 Sentinel 的紅旗清單與資安思考。
- **出去之後**：PRD 完成 → §14 交棒 → Compass 接手 DoR/實作/DoD。

## 三級制縮放

- 🟢 輕：改版微調 → 4 節（00/01/06/11）
- 🟡 中：一個功能模組 → 約 9 節
- 🔴 重：新產品、含金流/PII/權限 → 00–14 全程，§06 驗收與 §08 資安不可跳。
